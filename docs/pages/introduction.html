Introduction

MojoBlog is a simple, easy to use and quick blogging engine for MojoMotor. It integrates directly into the MojoMotor experience, allowing you to edit your blog content with MojoMotor's WYSIWYG editor and inline on your website. It features a range of powerful template tags that let you display your blog content easily and quickly.

Requirements

MojoBlog requires a licensed copy of MojoMotor 1.1.0+ to run. While an effort to retain backward compatibility will be made, you are always recommended to upgrade to the most recent version of MojoMotor.

Installation

Installing MojoBlog is just like any other MojoMotor addon. Simply drag the blog/ folder found in your download into the system/mojomotor/third_party/ folder. Ensure that your config/config.php and config/routes.php files are set to have 0777 permissions. Then, point your web browser to http://domain.com/index.php/admin/addons/blog/install and MojoBlog will install itself.

It's that simple - MojoBlog is installed!

A note on routes

For some of MojoBlog's functionality, such as single entry pages and pagination, a change to MojoMotor's routing file must be made. The installer will try to make this change automatically, however, in the event that this proves impossible, you'll need to manually add in the following lines to the bottom of the system/mojomotor/config/routes.php file:

$route = array(	'(.+)/entry/(.+)/?' => 'page/content/$1' ) + $route;
$route = array(	'(.+)/p/([0-9]+)/?' => 'page/content/$1' ) + $route;

Update

MojoBlog contains a built-in updater to make updating to the latest version effortless. Remember to backup your files, and then copy across the blog/ folder found in your download into the system/mojomotor/third_party/ folder. Then, point your web browser to http://domain.com/index.php/admin/addons/blog/update and MojoBlog will update itself.

Updating from MojoBlog 1.x to 2

A strong effort has been made to ensure backward compatibility. However, version 2 removes the concept of different blogs, so all your entries will be brought into a single blog. Additionally, Outfielder metadata has been removed, so any metadata attached to your entries will be lost during the upgrade process.
